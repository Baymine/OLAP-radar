## 2. Paper Snapshot

- **Title:** _Bespoke OLAP: Synthesizing Workload-Specific One-size-fits-one Database Engines_  
- **Authors:** Johannes Wehrstein, Timo Eckmann, Matthias Jasny, Carsten Binnig  
- **Publication date:** 2026-03-02  
- **Research problem:** Modern OLAP systems pay unavoidable overhead for generality—runtime schema handling, abstraction layers, and support for arbitrary workloads—even when production deployments often serve relatively stable analytical workloads. Building a hand-tuned engine for each workload is impractical due to engineering cost.  
- **Core method:** The paper presents a fully autonomous synthesis pipeline that uses LLM-driven code generation plus iterative validation and performance feedback to construct a workload-specific OLAP engine from scratch. The synthesis process co-designs storage structures and query execution, refining the generated system until it achieves correctness and high performance.

## 3. Why It Matters for OLAP

For OLAP practitioners, this paper points toward a new optimization frontier: not just optimizing queries within an engine, but synthesizing the engine itself around the workload. That directly affects **query execution**, **storage layout specialization**, and **benchmark-oriented system design**, especially for environments with repetitive dashboards, fixed marts, embedded analytics, or tenant-specific workloads. If the approach holds up, it could shift some analytical system design from manual kernel engineering toward automated workload compilation, complementing trends already seen in vectorized execution, code generation, and lakehouse physical optimization. It also raises practical research questions around maintainability, robustness, workload drift, and how such bespoke engines compare under evolving schemas and mixed workloads.
