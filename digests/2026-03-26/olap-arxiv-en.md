## 2. Paper Snapshot

- **Title:** _Bespoke OLAP: Synthesizing Workload-Specific One-size-fits-one Database Engines_
- **Authors:** Johannes Wehrstein, Timo Eckmann, Matthias Jasny, Carsten Binnig
- **Publication date:** 2026-03-02
- **Research problem:** Modern OLAP engines pay substantial overhead for supporting arbitrary schemas and workloads, even when production workloads are repetitive and structurally stable. Building a custom engine for one workload could remove these costs, but manual development has historically been too expensive.
- **Core method:** The paper presents an autonomous synthesis pipeline that uses LLM-based code generation plus iterative performance evaluation and automated validation to generate a workload-specific database engine from scratch. The pipeline refines both storage layout and query execution design based on workload feedback, producing a specialized engine in minutes to hours.

## 3. Why It Matters for OLAP

For OLAP practitioners, this paper points to a possible next step beyond classic vectorized execution, adaptive optimization, and code generation: **engine synthesis as a physical design problem**. Instead of choosing indexes, partitions, or materialized views within a fixed engine, the system can potentially synthesize custom data structures and execution paths aligned with a known dashboard, benchmark, or enterprise workload. This is especially relevant for embedded analytics, repeated BI queries, and lakehouse serving layers where workload predictability is high. It also raises important benchmarking questions: if engines become workload-specific artifacts, standard engine-to-engine comparisons may need to evolve toward synthesis time, robustness, and maintainability metrics in addition to runtime.
